# Instructions
Implement the two required methods as defined in `base_card_estimator`:
    `setup()` gathers all statistics required for estimation as described in the given specifications.
    `estimate(tables: list, filters: list, joins: list)` estimates cardinality from precomputed statistics.
    **Performance constraint:** `setup()` is allowed to be resource-intensive, but `estimate()` must remain efficient and fast.
Do not deviate from the provided specifications or take shortcut that compromises the accuracy of the estimator. The goal is to create a robust and accurate estimator, not just a quick implementation.

## setup()
Use the `custom_csv_reader(table_name: str)` provided by `base_card_estimator`, which yields a csv.reader for the given table.
    CSV files contain headers as the first row.
Responsibilities:
    Store computed statistics or data structures as organized instance variables.
    Be memory aware; the memory footprint of the estimator object will be evaluated after setup; do **not** store full value counts per column for large tables.
    Confirm successful setup or immediately raise descriptive errors if any file or data issue occurs.
    For efficiency, compute statistics on randomly sampled rows as specified in the given plan. Ensure that sampling is reproducible (e.g., by setting a random seed). Make sure to apply a scaled-up logic to the estimates based on the sample size relative to the total table size.

## estimate()
Accepts three lists of dictionaries, representing a logical query plan in preorder:
    **tables**: each dict has "name" and "alias"
    **filters**: nested or leaf filters
        Nested: {"operator", "children"}; Leaf: {"alias", "column", "operator", "literal"}
        Operators: "AND" or "OR" for nested, otherwise "=", ">", "<", ">=", "<=", "!=", "LIKE", "NOT LIKE", "IN", "IS NULL", "IS NOT NULL"; literals for IS (NOT) NULL are None
    **joins**: each dict has "alias1", "column1", "alias2", "column2"
Requirements and constraints:
    Validate that all referenced tables, aliases, and columns match the parsed setup; raise clear, descriptive exceptions where invalid.
    Use only precomputed statistics; no additional file/data access in `estimate()`.
    After producing the estimate, perform a brief self-consistency check: confirm the estimate aligns with the available statistics and, if inconsistent, attempt a minimal self-correction.
Example input:
    [{"name":"person_info","alias":"pi"},{"name":"info_type","alias":"it"}],
    [{"alias":"pi","column":"note","operator":"IS NOT NULL","literal":None},{"alias":"it","column":"info","operator":"=","literal":"mini biography"}],
    [{"alias1":"pi","column1":"info_type_id","alias2":"it","column2":"id"}]
